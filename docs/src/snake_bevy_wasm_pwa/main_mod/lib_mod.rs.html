3//! This module is like a lib.rs module for a binary CLI executable.
4//! The `lib_mod.rs` must not contains any input/output interface stuff.
5//! So the program logic can be separate from the interface.  
6
7// The `main_mod.rs` contains all input/output interface stuff.
8// This `lib_mod.rs` can then be used as dependency crate for other projects.
9
10// The `lib_mod.rs` does not have any real code. All the code is in modules in separate files.
11// The `lib_mod.rs` has just the list of modules, it publishes module's functions or class for the caller
12// and it has some global stuff like the Error enum.
13
14// The `main_mod.rs` uses the `anyhow` error library.
15// The `lib_mod.rs` uses the `thiserror` library.
16
